Phase 2

Plan Mode — 7 Scenarios

5:00 → 30:00

Use Plan Mode (Shift+Tab) to create detailed implementation plans for 7 wildly different scenarios. Each one demonstrates Copilot CLI's ability to research, reason, and plan across any discipline. We don't execute yet — just plan.

🎯 Why Plan First?

This mirrors real-world best practice: understand the problem before writing code. Plan mode forces Copilot to research, ask questions, and create a structured plan with trackable todos — before a single file is touched. This is the workflow for complex tasks.

UI / UX
🎨 Responsive Dashboard

Create a responsive analytics dashboard with dark/light mode, data visualization components, mobile-first layout, and accessibility compliance (WCAG AA).

Plan React / HTML
Data Engineering
📊 ETL Pipeline

Design a data pipeline that ingests CSV/JSON data, transforms it with validation rules, and outputs clean datasets. Include schema validation, error handling, and summary reporting.

Plan Python / SQL
Project Management
📋 Issue Triage Bot

Build a GitHub Actions workflow that auto-labels, prioritizes, and assigns incoming issues based on content analysis. Include templates, SLA tracking, and weekly digest generation.

Plan Actions / YAML
Infrastructure
🏗️ IaC Deployment

Create infrastructure-as-code templates for a multi-tier web app: networking, compute, storage, monitoring, and CI/CD pipeline. Target Azure or AWS with best practices baked in.

Plan Bicep / Terraform
Greenfield
🚀 Full-Stack App from Scratch

Spin up a complete app — API, database, auth, frontend, tests, docs, CI/CD — from a one-line description. Zero to deployed in one session. Show the power of starting from nothing.

Plan Full Stack Wow Factor
Brownfield / Modernization
🔍 Legacy Codebase Analysis

Point Copilot at an existing repo and ask it to analyze architecture, identify tech debt, suggest modernization paths, and create a prioritized migration plan. Research-heavy — no code changes.

Plan Analysis
Extensibility
🔌 Plugin + Agent + Skill + Hooks

Create a Copilot CLI plugin that includes a custom agent, a skill (SKILL.md), agent hooks for governance, and MCP server integration. The "full extensibility stack" in one project.

  • Keep each plan creation to ~3 minutes. Read the plan aloud, point out the SQL todos and dependencies

Phase 3

Execute — Parallel Sessions

30:00 → 45:00

Start all 7 plans executing simultaneously. While they run autonomously, use the time to demo key features, switch between sessions, and show the real-time power of the CLI.

🚦

Launch All Sessions

Switch to each session and tell Copilot to execute the plan. Use /autopilot for full autonomy or just say "start implementing" and let it run. Seven sessions, all building in parallel.

# Switch to each session and kick off execution $ ghcs --resume session-id-1 # UI Dashboard > start implementing $ ghcs --resume session-id-2 # ETL Pipeline > /autopilot start implementing
Live Autopilot Wow Factor

📝 Presenter Notes
  • Have a second terminal open to monitor file system changes in real time: watch -n 2 'find . -name "*.ts" -newer /tmp/mark | head -20'
  • If a session finishes early, great — show the completed output and commit history
  • If a session gets stuck, that's also a teachable moment — show how to intervene, adjust, and retry
  • For fleet mode, if you don't have fleet access, explain the concept and show the config flags
  • Watch the clock — don't linger on any one session too long
  • Build energy here: "While we were talking, Copilot has been coding across 7 projects simultaneously"
Phase 4

Grand Finale — Live Update

45:00 → 60:00

The showstopper. Create a new Copilot CLI session that evaluates all 7 completed sessions, generates a results page on this very reference site, pushes it live, and the audience watches the page appear in real-time.

🧠

Session Evaluation Agent

Start a new session. Ask Copilot to query its session store, analyze all 7 session outputs, and create a "Demo Results" page on the reference site. This is Copilot evaluating its own work.

$ ghcs > Query all sessions from today's demo. Summarize what each one accomplished — files created, lines of code, tests passed, etc. Create a new page on the copilot-cli-reference site called demo-results.html that shows all results. Push it live.

Checklist

Display Settings

  • Browser tab: Reference site loaded

  • Notifications silenced

📁

Working Directory

  • Create a clean demo directory: mkdir ~/copilot-demo && cd ~/copilot-demo
  • Pre-clone reference repo if needed: gh repo clone htekdev/copilot-cli-reference
  • Ensure plenty of disk space for 7 concurrent projects

Backup Plan

  • If a session hangs: Ctrl+C → retry or skip to next
  • If auth fails: have a PAT ready as backup
  • If time runs short: skip to Phase 4 — the finale is the most impactful moment
